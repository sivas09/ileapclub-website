# Agenda Generation Design

## Purpose

This document defines the future iLEAP Club meeting agenda system for the member portal.

The old Drupal meeting module and sample RTF agenda are reference material only. They show a fixed placeholder pattern such as:

```text
role1_uid
role2_uid
role3_uid
role4_uid
role5_uid
role6_uid
```

The new system should not copy that fixed design. The new design uses flexible agenda templates, sections, and role slots so Junior, Senior, Debate, Town Hall, Competition, and Special Event meetings can all be supported cleanly.

## Design Goals

- Support many meeting formats.
- Avoid fixed role columns.
- Let each template define its own sections and role slots.
- Allow members to claim roles.
- Prevent duplicate claims for the same role slot.
- Support facilitator/admin override.
- Support role locking and reopening.
- Generate clean RTF agendas.
- Show open roles clearly.
- Support speaker/evaluator pairings.
- Keep facilitator-only notes separate from student-visible instructions.

## Agenda Template Types

Initial template types:

```text
Junior Regular Meeting
Senior Regular Meeting
Debate Meeting
Town Hall Leadership Challenge
Competition Meeting
Special Event
```

## Template Structure

```text
agenda_templates
  agenda_sections
    agenda_role_slots
```

An agenda template is the overall meeting format.

An agenda section is one part of the meeting, such as:

- Arrival and agenda changes
- Introduction
- Prepared speeches
- Debate round
- Table topics
- Town hall discussion
- Evaluations
- Awards and closing

An agenda role slot is a claimable or assignable role inside a section.

## Section Requirements

Each section should support:

- section title
- section order
- start time
- duration
- public notes
- facilitator-only notes
- student-visible instructions
- role slots

## Role Slot Requirements

Each role slot should support:

- role name
- slot label
- slot order
- duration
- required/optional flag
- self-claim allowed flag
- assigned member
- pairing with another slot
- student-visible instructions
- facilitator-only notes

## Speaker and Evaluator Pairing

The system should support pairings such as:

```text
Speaker 1 -> Evaluator 1
Speaker 2 -> Evaluator 2
Debate Lead -> Debate Rebuttal
```

This should be stored as a relationship between agenda role slots, not as hardcoded columns.

## Role Claiming Rules

Members should be able to claim roles for upcoming meetings.

Rules:

- Each member should choose at least two roles when enough roles are available.
- A role slot can only be claimed by one member.
- Members can claim only roles in their own club meetings.
- Locked roles cannot be claimed.
- Facilitator/Admin can override assignments.
- Facilitator/Admin can lock roles before a meeting.
- Facilitator/Admin can reopen roles if needed.

The backend must use a transaction or unique constraint so two members cannot claim the same role slot at the same time.

## RTF Agenda Download

Future endpoint:

```text
GET /api/meetings/:id/agenda.rtf
```

Allowed users:

- Student/Member in the meeting club
- Parent linked to a student in the meeting club
- Facilitator assigned to the club
- Admin

## RTF Agenda Content

The generated RTF should include:

- iLEAP Club Meeting Agenda
- Club name
- Centre
- Meeting date
- Session time
- Location or online link
- Agenda sections
- Role names
- Assigned member names
- Open roles shown as `Open`
- Speaker/evaluator pairings
- Footer: `Generated by iLEAP Club Member Portal`

## Format Improvements Over Old Drupal Agenda

The new RTF should improve readability through:

- cleaner heading hierarchy
- consistent fonts
- clear section titles
- tables for role assignments
- visible open roles
- structured speaker/evaluator pairings
- better spacing
- support for multiple agenda formats
- no fixed `role1_uid` style placeholders

## MVP Recommendation

Build first:

1. Agenda templates
2. Agenda sections
3. Agenda role slots
4. Meeting role generation from template slots
5. Member role claiming
6. Facilitator/Admin override
7. Role locking and reopening
8. RTF agenda generation

Delay visual drag-and-drop agenda editing until after the basic workflow works.
